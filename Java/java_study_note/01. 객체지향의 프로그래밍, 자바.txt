
* 자바의 특징

1. 객체지향 언어
	객체지향: 여러 덩어리들이 모여서 하나의 프로그램을 제작, 협업성 뛰어남
2. 플랫폼 독립적
	어디에서든지 
3. 멀티스레드 지원
	멀티태스킹이 가능
	하나의 프로그램이 여러가지 작업을 처리 
4. 간단한 코드
	객관적인 관점으로서 장점
	섬세한 코딩이 다소 불편하다는 단점
5. 뛰어난 보안성
6. 고성능
7. 모듈화
	코드같은 것을 가두어 두는것




<절차 지향 vs 객체 지향>

1. 절차 지향 프로그래밍
- ex) 	
	자동차를 만들 때 전체를 하나의 몸체로 만든다.
	바퀴, 엔진, 핸들... 등이 하나로 뭉쳐져 있다.
- 단점
	a. 한 부분이 고장나면 전체가 동작하지 않는다. 
	b. 그리고 유지 보수에 어려움이 있다.
	c. 개발 기간이 오래 걸리고 비용이 많이 든다.
	d. 협업이 어렵다.
- 장점
	a. 섬세한 코딩 가능하다.
	b. 메모리 사용이 적다.
	c. 작은 규모의 프로그램에 적합하다.


2. 객체 지향 프로그래밍
- ex) 
	자동차를 만들 때, 부품별로 미리 만들어 두고, 나중에 한번에 조립하는 형태이다.
- 장점
	a. 한 부분이 고장나도 다른 부분 동작 가능하다.
	b. 유지 보수가 쉽다.
	c. 개발기간이 짧고 비용도 적게 든다.
	d. 협업이 가능하다.
- 단점
	a. 메모리의 사용이 좀 더 많다.
	b. 불필요한 부분까지 프로그램에 포함된다.
	c. 대규모의 프로그램에 적합하다.
